Why Learn Programming?

Programming is a tool

A spreadsheet is great for small, clean tables. Programming helps when data becomes large, repetitive, messy, or needs to connect with sensors, cameras, cloud files, or AI models.

In research and engineering, the exact tool you need often does not already exist. Programming lets you build it.

Greenhouse examples

  • Read 10,000 sensor measurements.
  • Remove impossible values.
  • Plot temperature over time.
  • Alert when humidity is too low.
  • Train an AI model to classify plant images.

What Is a Program?

A program is a set of instructions that tells the computer what to do next.

Programming is not magic. It is giving clear instructions, in the right order, using a language the computer can run.

Human recipe

  1. Measure water temperature.
  2. Compare with target range.
  3. If too hot, turn on cooling.
  4. If normal, keep monitoring.
Timing: 2 minutes. Ask: what happens if recipe steps are out of order?

Conditionals: If This, Then That

temperature_c = 29

if temperature_c > 28:
    print("Too hot: turn on cooling fan")
else:
    print("Temperature is okay")

An if statement lets the program choose different actions depending on the situation.

Indentation matters: the indented lines belong inside the condition.

Timing: 4 minutes. Emphasize colon and indentation. Let them break indentation intentionally.

Greenhouse Decision Example

temp_c = 25
humidity = 45

if temp_c > 28 and humidity < 40:
    print("Hot and dry: check irrigation and cooling")
elif temp_c > 28:
    print("Hot: turn on cooling")
elif humidity < 40:
    print("Dry: consider misting")
else:
    print("Climate looks stable")
Use and when both conditions must be true. Use or when at least one condition can be true.

Useful List Operations

temperatures = [23.1, 23.4, 24.0, 24.8, 25.2]

print("Number of readings:", len(temperatures))
print("Minimum:", min(temperatures))
print("Maximum:", max(temperatures))

Loops: Repeat Without Rewriting

temperatures = [23.1, 23.4, 24.0, 24.8, 25.2]

for temp in temperatures:
    print("Reading:", temp)

A loop repeats the same action for each value in a collection.

Without loops, analyzing thousands of sensor readings would be impossible by hand.

Timing: 4 minutes. Trace the loop line by line.

Loop + Condition

temperatures = [23.1, 29.2, 24.0, 30.1, 25.2]

for temp in temperatures:
    if temp > 28:
        print(temp, "is too hot")
    else:
        print(temp, "is okay")
Challenge: Count how many readings are too hot.
Timing: 4 minutes. Let advanced students add a counter variable.

Counting Events

temperatures = [23.1, 29.2, 24.0, 30.1, 25.2]

hot_count = 0

for temp in temperatures:
    if temp > 28:
        hot_count = hot_count + 1

print("Number of hot readings:", hot_count)
A counter variable starts at zero and increases when a condition is met.
Timing: 3 minutes. Explain variable update carefully.

Functions: Reusable Code Blocks

def c_to_f(temp_c):
    temp_f = temp_c * 9/5 + 32
    return temp_f

print(c_to_f(25))
print(c_to_f(30))

A function packages code so we can use it again and again.

Functions make programs easier to test, read, and improve.

Timing: 4 minutes. Define input/parameter and return value.

Function Example: Sensor Voltage to Light

# Example conversion for a PAR light sensor
# Suppose calibration says: PPFD = voltage_mV * 0.8

def voltage_to_ppfd(voltage_v):
    voltage_mV = voltage_v * 1000
    ppfd = voltage_mV * 0.8
    return ppfd

print(voltage_to_ppfd(0.5))
print(voltage_to_ppfd(1.2))
This mirrors real sensor work: raw electrical signal → calibrated scientific measurement.
Timing: 4 minutes. Connect to Arduino sensor lessons from Day 2.

Analyze a Small Greenhouse Dataset

import pandas as pd
import matplotlib.pyplot as plt

df = pd.DataFrame({
    "time_h": [0, 1, 2, 3, 4, 5, 6],
    "temp_c": [22.8, 23.0, 23.7, 24.5, 25.8, 28.4, 29.1],
    "humidity": [64, 62, 59, 56, 50, 42, 38]
})

print("Average temperature:", df["temp_c"].mean())
print("Lowest humidity:", df["humidity"].min())

plt.plot(df["time_h"], df["temp_c"], marker="o", label="Temperature")
plt.xlabel("Time (hour)")
plt.ylabel("Temperature (°C)")
plt.title("Greenhouse Climate Trend")
plt.legend()
plt.show()
Timing: 6 minutes. This is the main live coding example near the end.

Errors Are Feedback

Common beginner errors

  • Missing quotation mark
  • Wrong indentation
  • Using = instead of ==
  • Forgetting to convert input text to number
  • Misspelling a variable name

Debugging mindset

Do not panic. Read the error message, find the line number, check one thing at a time, and run again.

Programming is mostly testing, fixing, and improving.

Timing: 3 minutes. Demonstrate a simple NameError or SyntaxError if time allows.

Debugging Demo

# Broken code
temperature = "25"
print(temperature + 2)
# Fixed code
temperature = "25"
temperature = float(temperature)
print(temperature + 2)
The error happens because Python cannot add text and a number. Convert the text first.
Timing: 3 minutes. Run broken code first, then fixed code.
